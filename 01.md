# Januscape 关键背景分析

## 问题 1：内核是否已经完成该问题的代码合入？

结论：上游 Linux 主线已合入。

依据：
- 上游修复提交已存在：81ccda30b4e8
- 提交标题：KVM: x86: Fix shadow paging use-after-free due to unexpected role
- 提交链接：https://github.com/torvalds/linux/commit/81ccda30b4e8

## 问题 2：Ubuntu 22 和 24 的标准 Livepatch 流程是否已包含这个修复？

结论：截至当前时间点（2026-07-09），不能认为已包含。

依据：
- 在当前测试主机执行 `pro fix CVE-2026-53359`，返回：
  - no fix is available yet
  - linux 与 linux-hwe-6.17 仍受影响
- Ubuntu CVE 页面（CVE-2026-53359）对 Jammy/Noble 多条内核线仍显示 Vulnerable。

说明：
- 上游已修复，不等于 Ubuntu 各发行版内核线与 Livepatch 通道已同步覆盖。

## 问题 3：如果 Ubuntu 还没有修复，是否只能自己按标准 Livepatch 流程尝试修复？

结论：可以且应当这样做，但应作为临时过渡方案。

建议执行顺序：
1. 先做立即缓解：若业务允许，优先关闭 nested virtualization，降低风险面。
2. 在测试环境按标准流程实施自定义 livepatch：构建、加载、验证、回归。
3. 持续跟踪 Ubuntu 官方 USN/LSN 与 CVE 状态。
4. 官方修复发布后尽快切回官方补丁并安排重启生效。
